# Stage C: Pseudo-Labeling

Train a classifier on real scores from Stage B, evaluate on real test scores, and pseudo-label synthetic samples.

## Quick Start

**Using configuration files (recommended):**

Train classifier:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c --config configs/stage_c_train.yaml
```

Test classifier:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c --config configs/stage_c_test.yaml
```

Label synthetic data:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c --config configs/stage_c_labeling_syn.yaml
```

**Using CLI arguments:**

Train:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c train \
  --dataset RAPzs \
  --prompting fixed-rule \
  --score_name blip \
  --strategy identity \
  --threshold 0.5 \
  --classifier bayes \
  --bayes_mode gauss \
  --lora_dir rapzs_lora_4_with_transform_256_192_loss_train_model_21
```

Train with custom dataset paths:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c train \
  --dataset RAPzs \
  --prompting fixed-rule \
  --score_name blip \
  --strategy identity \
  --threshold 0.5 \
  --classifier bayes \
  --bayes_mode gauss \
  --lora_dir rapzs_lora_4_with_transform_256_192_loss_train_model_21 \
  --path_dataset "/custom/dataset/" \
  --path_gt "/custom/dataset_all.pkl" \
  --path_gt_img "/custom/images/"
```

Test:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c test \
  --dataset RAPzs \
  --prompting fixed-rule \
  --score_name blip \
  --strategy identity \
  --threshold 0.5 \
  --classifier bayes \
  --bayes_mode gauss \
  --lora_dir rapzs_lora_4_with_transform_256_192_loss_train_model_21
```

Label synthetic:
```bash
python -m src.stage_c_pseudolabeling.run_stage_c labelingSyn \
  --dataset RAPzs \
  --prompting fixed-rule \
  --score_name blip \
  --strategy identity \
  --threshold 0.5 \
  --classifier bayes \
  --bayes_mode gauss \
  --lora_dir rapzs_lora_4_with_transform_256_192_loss_train_model_21
```

## Configuration Files

See `configs/stage_c_train.yaml`, `configs/stage_c_test.yaml`, and `configs/stage_c_labeling_syn.yaml` for examples.

```yaml
mode: "train"  # or "test" or "labelingSyn"

dataset: "RAPzs"
prompting: "fixed-rule"
score_name: "blip"
strategy: "identity"

# Optional custom dataset paths (uncomment to use)
# dataset:
#   path_dataset: "/path/to/custom/dataset/"
#   path_gt: "/path/to/custom/dataset_all.pkl"
#   path_gt_img: "/path/to/custom/images/"

classifier: "bayes"
bayes_mode: "gauss"
# Optional classifier parameters
# prior_pos: 0.5           # Bayes prior for positive class (0-1)
# kde_bandwidth: "scott"   # KDE bandwidth (float or 'scott')
threshold: 0.5

lora_dir: "rapzs_lora_4_with_transform_256_192_loss_train_model_21"
labeling_policy: "hardlabelling"

random_state: 42
val_frac: 0.2
```

## Inputs

- Stage B real scores in `<dataset>_<prompting>_<score_name>_<strategy>_scores/`
  - `scores_train.xlsx`: Training set scores
  - `scores_test.xlsx`: Test set scores
- Stage B synthetic scores in same folder (generated by Stage B)

## Outputs

- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/classifier.pkl`: Trained classifier
- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/classifier_tag.txt`: Classifier tag
- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/train_predictions.csv`: Predictions on real training data
- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/val_predictions.csv`: Predictions on validation split
- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/test_predictions.csv`: Predictions on real test data
- `<dataset>_<prompting>_<score_name>_<strategy>_<classifier>_si/artifacts/pseudolabels_syn.csv`: Pseudo-labels for synthetic data

## Modes

- `train`: Train classifier on real training scores, split into train/val
- `test`: Evaluate trained classifier on real test scores
- `labelingSyn`: Use trained classifier to pseudo-label synthetic dataset

## Classifiers

- `threshold`: Simple threshold-based classifier
- `bayes`: Bayesian classifier with density estimation
  - `gauss`: Gaussian density model
  - `kde`: Kernel density estimation

## Key Parameters

- `--dataset`: Dataset name (PA100k, PETA, PETAzs, RAPv1, RAPv2, RAPzs)
- `--prompting`: Prompting type descriptor
- `--score_name`: Scoring model name (blip)
- `--strategy`: Strategy for attribute handling (identity)
- `--threshold`: Decision threshold for labeling (default: 0.5)
- `--classifier`: Classifier type (threshold or bayes)
- `--bayes_mode`: Bayes density model (gauss or kde)
- `--prior_pos`: Bayes prior for positive class (0-1)
- `--kde_bandwidth`: KDE bandwidth (float or 'scott')
- `--lora_dir`: Base directory for Stage B/C outputs
- `--labeling_policy`: Labeling policy (hardlabelling)
- `--random_state`: Random seed for reproducibility
- `--val_frac`: Fraction of training data for validation
- `--path_dataset`: Custom path to dataset images directory
- `--path_gt`: Custom path to ground truth pickle file
- `--path_gt_img`: Custom path to ground truth images directory

## Workflow

**Train:**
1. Load real training scores from Stage B
2. Split into 80% training and 20% validation
3. Fit classifier on training split
4. Evaluate on validation split
5. Save classifier to artifacts

**Test:**
1. Load trained classifier from artifacts
2. Load real test scores from Stage B
3. Make predictions on test set
4. Evaluate and save results

**Label Synthetic:**
1. Load trained classifier from artifacts
2. Load synthetic scores from Stage B
3. Make predictions for each synthetic sample
4. Generate pseudo-label vectors based on predictions
5. Save results to CSV

## Understanding the Output

- **train_predictions.csv**: Training set predictions with:
  - `pos/neg`: Whether sample is positive or negative in training split
  - `score`: Score value from Stage B
  - `decision`: Binary decision (0 or 1)
  - Per-attribute columns with pseudo-labels

- **test_predictions.csv**: Test set predictions with same structure

- **pseudolabels_syn.csv**: Synthetic data predictions with pseudo-labels for all attributes

- **JSON metrics files**: Pedestrian metrics computed on predictions:
  - Per-attribute accuracy
  - Overall accuracy (micro and macro)
  - Per-attribute prediction counts
